# 1. 자료구조와 알고리즘
- 자료 구조: 자료(data) + 구조(저장공간(memory) + 연산(읽기,쓰기,삽입,삭제,탐색))
- 알고리즘: 입력 데이터 > 유한한 횟수의 연산들의 반복 > 정답

>> 자료구조와 알고리즘은 서로 분리가 불가능한 관계

# 2. 알고리즘
- 인류 최초의 알고리즘: 유클리드의 최대공약수 계산 알고리즘(GCD / 1_GCD.py)
- gcd(8|2) = max{1,2,4} = 4
- 처음에는 A에서 B를 빼는걸 반복했다. 그러나, 빼기가 아닌 나누기를 사용하면 더 빠르게 최대공약수를 구할 수 있다.

# 3. 가상 컴퓨터 + 가상 언어 + 가상 코드
- 자료구조와 알고리즘은 코드들(C, java, Python, 등)로 구성되어, 컴퓨터에서 작동한다. 그러나, 언어마다. 컴퓨터마다 속도가 상이하다. > 속도에 대한 객관적 평가가 불가능
- 가상 컴퓨터(virtual machine) + 가상 언어(pseudo language) + 가상 코드(pseudo code)
- 기본 연산: 1단위 시간 내에서 수행 가능한 연산들
  1. 배정, 대입, 복사 연산
  2. 산술 연산
  3. 비교 연산
  4. 논리 연산
  5. 비트 연산

- 가상 컴퓨터에서 가상 코드와 가상 언어. 그리고, 기본 연산들을 통해 나오는 시간을 알고리즘 성능 비교의 척도로 사용 한다.
- 가상 언어에선 '기본 연산', '비교 연산(if문)', '반복 연산', 함수(함수 정의, return, 등)로 구성된다.

# 4. 알고리즘의 시간 복잡도(time Complexity)
- 모든 입력에 대해 기본 연산 횟수를 더한 후의 평균... 이 가장 이상적인 알고리즘의 시간이지만, 현실적으로 불가능
- 가장 안좋은 입력(worstCase input)에 대한 '기본 연산 횟수를 측정'하여 사용.

  >> 'worstCase input'을 사용 할 경우, 그 어떠한 입력도 'worstCase input'보다 수행 시간이 크지 않다라는 장점이 있음.

  >>>> 그러므로 '알고리즘 수행시간'은 ''worstCase input'에 대한 기본 연산 횟수'로 정의한다.

ex) for i in range(n):
      print(i)

- 위의 예시(알고리즘)은 N번 수행된다. = BigO(N)

# 5. 빅오(big-O) 표기법
- 빅오 표기법: 함수 값을 결정하는 최고차항만으로 간단하게 표기

ex) for i in range(n):
      print(i)
      print(i+i)

- 위의 예시는 출력 연산, 출력 연산, 사칙 연산. 총 3번의 기본 연산을 n번 수행하는 알고리즘이다. 즉, 3N이라는 시간이 걸리지만, 빅오 표기법에선 3N이 아니라, 최고차항인 N으로 표시
- 결론: 최고차항만 남긴다 > 최고차항의 계수는 생략한다. > 'Big-O(최고차항)'으로 표기한다.
- 시간 복잡도에는 크게 3가지가 존재한다. Big-O(n), Big-O(log2n), Big-O(n*n). 이 세가지이다. 속도는 log2n이 가장 빠르며, n제곱이 가장 속도가 느리다.
